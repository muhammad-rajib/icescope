# Automatic Updates

IceScope is prepared for Tauri updater support, but automatic updates should only be enabled after release signing is configured.

## Required pieces

To enable safe automatic updates, IceScope needs:

- Tauri updater plugin in Rust and frontend dependencies.
- A signing key pair generated by the Tauri CLI.
- Public updater key in `src-tauri/tauri.conf.json`.
- Private signing key stored only in GitHub Actions secrets.
- A stable update endpoint such as a `latest.json` file attached to GitHub Releases.
- App startup or Settings-page update checks.

## Recommended setup

1. Install updater packages.
2. Generate updater signing keys.
3. Add the public key and endpoint to Tauri config.
4. Add private key secrets to GitHub.
5. Update the release workflow to sign bundles and publish updater metadata.
6. Add app code to check for updates after startup and from **Settings → Check for updates**.

## Example config shape

```json
{
  "plugins": {
    "updater": {
      "pubkey": "PASTE_PUBLIC_KEY_HERE",
      "endpoints": [
        "https://github.com/muhammad-rajib/icescope/releases/latest/download/latest.json"
      ],
      "windows": {
        "installMode": "passive"
      }
    }
  }
}
```

## GitHub Actions secrets

Add these repository secrets before enabling signed updates:

- `TAURI_SIGNING_PRIVATE_KEY`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

Never commit the private key to the repository.

## Release rule

Automatic updates depend on semantic versioning. Every public release must update:

- `package.json`
- `Cargo.toml`
- `src-tauri/tauri.conf.json`
- `CHANGELOG.md`

Use `scripts/release.sh` to keep versions synchronized.

## Current status

Updater support is **planned**. Do not turn it on for public users until:

- macOS and Windows signing are ready.
- Updater signing secrets exist in GitHub.
- A test release successfully updates from an older installed version.
